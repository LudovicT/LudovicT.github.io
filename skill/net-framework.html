.NET Framework

Développement de batch, applications, site web et autres type de programmes.

Le framework .NET est une plateforme de développement largement utilisée pour la création d’application dans les environnements Microsoft (Windows, Windows Store, Windows Phone, Windows Server et Microsoft Azure). Le framework comprend notamment le langage C# dont il est spécifiquement question dans cette compétence. 

Je m’y suis intéressé dès le troisième semestre de ma formation à IN'TECH car je devais réaliser un projet en utilisant ce langage. J’ai alors réalisé un projet en C# avec Windows Forms pour pouvoir faire un scanner réseau. 

C# 

Ce langage que l’on compare souvent au Java est orienté objet. C’est pour moi un langage moderne, jeune, qui a été conçu pour être ce qu’il est aujourd’hui. L’écosystème mis en place par Microsoft participe à son succès.

ASP.NET MVC

ASP.NET est la plateforme de développement de Microsoft permettant la réalisation d'applications web.

J’ai commencé à utiliser ASP.NET MVC à Candriam pour le développement de l'IHM MasterPrice. J'ai d'ailleurs appris la technologie sur le moment grâce à un livre (ASP.NET MVC 4 : Développement d'applications en C#) aimablement prêté par un collègue. En une semaine j'ai acquis les connaissances nécessaires pour être opérationnel et je ne cesse de m'améliorer depuis.

Entity Framework

Cet ORM (Object Relation Model) créé par Microsoft a été le premier que j’ai utilisé. Plutôt simple à mettre en œuvre, il reste tout de même lourd. Ce framework propose quatre façons différentes de l’utiliser :  

Database first

Dans ce mode, le mapping objet se fait à partir du modèle de données directement en base de données. C’est la façon que j’utilise tout le temps car j’aime bien avoir la main sur ma base de données et j’utilise des procédures stockées pour exécuter des requêtes de modifications et de suppressions sur mes données. 

Model first

Celui-ci permet de « dessiner » son modèle de données graphiquement et, ainsi, d’initialiser sa base de données en plus de créer le mapping objet. Je ne l’ai jamais utilisé.  

Code first

Cette dernière façon de procéder permet d’initialiser sa base de données directement depuis le code C#. Il suffit de créer une classe, y insérer un ou deux attributs, pour que la table correspondante se crée. On peut, à partir de là, utiliser l’objet directement pour effectuer des opérations sur la base de données. Ça semble très sympathique mais cela devient vite difficilement à maintenir dans le temps même si des mécanismes de migrations de modèle existent. Je n’utilise pas du tout ce mode professionnellement. Je l’ai utilisé pour concevoir des PoC (Proof Of Concept) rapidement ou des projets personnels qui ne nécessitaient pas de moyens importants. 

Windows Forms 

Windows Forms est le nom de l'interface graphique qui est incluse dans framework .NET, fournissant l'accès via du Managed code à l'API Windows.

J'ai eu l'occasion d'utiliser WinForm en 3ème semestre à IN'TECH pour un projet informatique relativement simple.
Puis, en travaillant chez Candriam, j'ai eu l'occasion d'approfondir mes connaissances en travaillant sur l'IHM MasterFile qui est un client WinForm en VB.NET. J'ai donc eu l'occasion de voire une très grosse application avec du vécu, 8 ans à l'époque, et sur laquel j'ai eu beaucoup d'occasion de me familiarisé au cours des projets dans lesquels j'ai été impliqué (refactoring, correction, mise à jour, ...).

Windows Communication Foundation (WCF)

Windows Communication Foundation (WCF) est une infrastructure permettant de générer des applications orientées services. Avec WCF, vous pouvez envoyer des données sous forme de messages asynchrones d'un point de terminaison de service à un autre. Un point de terminaison de service peut faire partie d'un service disponible en continu et hébergé par IIS, ou il peut s'agir d'un service hébergé dans une application. Un point de terminaison peut être un client d'un service qui demande des données auprès d'un point de terminaison de service. Les messages peuvent être simplement constitués d'un caractère ou d'un mot unique envoyé au format XML, ou se présenter sous la forme d'un flux de données binaires plus complexe.

J'ai beaucoup utilisé WCF à Candriam, la directive des architectes de Candriam est d'utiliser WCF pour communiquer entre différentes équipes.
J'ai eu l'occasion de mettre à jour un service de 2 endpoints pour la migration de MasterPrice vers SQL Server, de créer un service de 2 endpoints pour la gestion du cache dans l'IHM MasterFile.
Mais aussi de procéder à des corrections et des évolutions sur les nombreux services, 6 services pour 11 endpoints, que l'équipe administre et maintient. Dans la plupart des services il y a deux endpoints, un standard et un paginé pour des récupérations massive de données.

Service Windows

Dans les systèmes d'exploitation de type Windows NT, un service (ou service Windows) est un programme qui fonctionne en arrière-plan. Il est similaire à un daemon d'Unix.

Dans le cadre du projet DataFeed de mise en cache de données pour l'IHM MasterFile j'ai mis en place un service Windows permétant de :

  • Catégoriser, enrichir et exécuter parallèlement des requètes via les composants DataFeed pointant vers Bloomberg
  • Gérer les excéptions et executer un fallback si possible
  • Stocker les données reçus dans notre base de donnée SQL Server sous forme clef/valeur pour un accès très rapide

J'ai également pu participer à la résolution de bug et a des améliorations sur les Service Windows composant le coeur de DataFeed, la récupération des données.

 

En conclusion, l’écosystème relatif au framework .NET est vaste. J’apprécie beaucoup d’évoluer dans cet environnement qui me paraît simple et cohérent. De plus, les outils mis à disposition par Microsoft comme Visual Studio participent grandement à cette sensation de bien-être.
